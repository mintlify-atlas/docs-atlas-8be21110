The JDBC specification acknowledges that some vendor-specific SQL may be required for certain database features.
To help developers write portable JDBC applications across multiple database products, JDBC defines a special escape syntax.
The driver translates these escape sequences into native PostgreSQL SQL.

## Enabling and disabling escape processing

Escape processing is enabled by default. You can disable it per statement:

```java
statement.setEscapeProcessing(false);
```

To preview how an escape sequence will be translated without executing it, use `Connection.nativeSQL()`:

```java
String nativeSql = connection.nativeSQL("SELECT {fn week({d '2005-01-24'})}");
// Returns: SELECT extract(week from DATE '2005-01-24')
```

## Example

The following two queries are equivalent. The first uses portable JDBC escape syntax; the second uses PostgreSQL-specific SQL:

<CodeGroup>

```java Portable (JDBC escape syntax)
ResultSet rs = st.executeQuery("SELECT {fn week({d '2005-01-24'})}");
```

```java PostgreSQL native SQL
ResultSet rs = st.executeQuery("SELECT extract(week from DATE '2005-01-24')");
```

</CodeGroup>

## Date and time escapes

The JDBC specification defines escapes for date, time, and timestamp literals.

| Escape | Translates to | Notes |
|---|---|---|
| `{d 'yyyy-mm-dd'}` | `DATE 'yyyy-mm-dd'` | Date literal |
| `{t 'hh:mm:ss'}` | `TIME 'hh:mm:ss'` | Time literal |
| `{ts 'yyyy-mm-dd hh:mm:ss.f...'}` | `TIMESTAMP 'yyyy-mm-dd hh:mm:ss.f'` | Fractional seconds are optional |

## LIKE escape character

You can specify which escape character to use in `LIKE` comparisons to protect the wildcard characters `%` and `_`:

```
{escape 'escape-character'}
```

The driver supports this only at the end of the comparison expression. For example, using `|` as the escape character to protect `_`:

```java
rs = stmt.executeQuery("select str2 from comparisontest where str1 like '|_abcd' {escape '|'} ");
```

## Outer join syntax

You can express outer joins using the following escape:

```
{oj table (LEFT|RIGHT|FULL) OUTER JOIN (table | outer-join) ON search-condition}
```

Example:

```java
ResultSet rs = stmt.executeQuery("select * from {oj a left outer join b on (a.i=b.i)} ");
```

## Scalar functions

Scalar functions are invoked with the escape syntax `{fn function_name(arguments)}`. The driver supports nesting and mixing of escaped functions and escaped values.

<Note>
  Some functions in the tables below are translated but reported as not supported because they duplicate or reorder their arguments. This is harmless for literal values or column references but will cause problems with prepared statements, because the translated SQL requires more parameters than the original.

  For example, `{fn right(?,?)}` translates to `substring(? from (length(?)+1-?))` — three bind placeholders instead of two.
</Note>
