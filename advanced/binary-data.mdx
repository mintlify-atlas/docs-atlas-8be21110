PostgreSQL provides two distinct ways to store binary data. Binary data can be stored in a table using the data type `BYTEA` or by using the Large Object feature, which stores the binary data in a separate table in a special format and refers to that table by storing a value of type `OID` in your table.

## Choosing between BYTEA and Large Objects

<Note>
Since driver version 7.2, `getBytes()`, `setBytes()`, `getBinaryStream()`, and `setBinaryStream()` operate on the `BYTEA` data type. In 7.1 and earlier, these methods operated on the `OID` type associated with Large Objects.
</Note>

| | BYTEA | Large Objects |
|---|---|---|
| Max size | Up to 1 GB per column | Effectively unlimited |
| Best for | Small to medium binary data | Very large files |
| Memory use | High for large values (entire value loaded) | Streamed; lower memory footprint |
| Transaction required | No | Yes — must call `setAutoCommit(false)` |
| Delete behavior | Deleted with the row | Must be deleted separately |
| Access control | Follows table permissions | Any connected user can view or modify any Large Object |

<Warning>
Deleting a row that contains a Large Object reference does **not** delete the Large Object itself. You must delete it separately. Additionally, any connected user can view or modify any Large Object regardless of row-level permissions — consider this before storing sensitive data as Large Objects.
</Warning>

## Working with BYTEA

Use the standard JDBC stream and byte-array methods to read and write `BYTEA` columns.

### Inserting binary data

```sql
CREATE TABLE images (imgname text, img bytea);
```

```java
File file = new File("myimage.gif");
try (FileInputStream fis = new FileInputStream(file);
     PreparedStatement ps = conn.prepareStatement("INSERT INTO images VALUES (?, ?)")) {
    ps.setString(1, file.getName());
    ps.setBinaryStream(2, fis, (int) file.length());
    ps.executeUpdate();
}
```

<Note>
The length parameter to `setBinaryStream` must be correct. There is no way to indicate that the stream is of unknown length. If you are in this situation, you must read the stream yourself into temporary storage, determine the length, and then send the data from temporary storage to the driver.
</Note>

### Retrieving binary data

```java
try (PreparedStatement ps = conn.prepareStatement("SELECT img FROM images WHERE imgname = ?")) {
    ps.setString(1, "myimage.gif");
    try (ResultSet rs = ps.executeQuery()) {
        while (rs.next()) {
            byte[] imgBytes = rs.getBytes(1);
            // use the data in some way here
        }
    }
}
```

You can also retrieve binary data as an `InputStream` by calling `rs.getBinaryStream(1)` instead of `rs.getBytes(1)`.

## Working with Large Objects

Large Objects are accessed through the standard JDBC `Blob` interface. Use `PreparedStatement.setBlob()` to insert and `ResultSet.getBlob()` to retrieve.

<Warning>
All Large Object API calls must be within a transaction block. Call `setAutoCommit(false)` before any Large Object operation and `conn.commit()` when done.
</Warning>

### Schema

```sql
CREATE TABLE imageslo (imgname text, imgoid oid);
```

### Inserting a Large Object

```java
// All LargeObject API calls must be within a transaction block
conn.setAutoCommit(false);

File inputFile = new File("myimage.gif");
// Now insert the row into imageslo
try (PreparedStatement ps = conn.prepareStatement("INSERT INTO imageslo VALUES (?, ?)");
     FileInputStream fis = new FileInputStream(inputFile)) {
    ps.setString(1, file.getName());
    ps.setBlob(2, fis, inputFile.length());
    ps.executeUpdate();
}

// Finally, commit the transaction.
conn.commit();
```

### Retrieving a Large Object

```java
// All LargeObject API calls must be within a transaction block
conn.setAutoCommit(false);

try (PreparedStatement ps = conn.prepareStatement("SELECT imgoid FROM imageslo WHERE imgname = ?")) {
    ps.setString(1, "myimage.gif");
    try (ResultSet rs = ps.executeQuery()) {
        while (rs.next()) {
            // Read all data at once
            byte[] contents = rs.getBytes(1);
            // Read all data as InputStream
            Blob blob = rs.getBlob(1);
            try (InputStream is = blob.getBinaryStream()) {
                // Process the input stream. The input stream is buffered, so you don't need to
                // wrap it in a BufferedInputStream
            } finally {
                blob.free();
            }
        }
    }
}

// Finally, commit the transaction.
conn.commit();
```

### Updating a Large Object

```java
// All LargeObject API calls must be within a transaction block
conn.setAutoCommit(false);

try (PreparedStatement ps = conn.prepareStatement("SELECT imgoid FROM imageslo WHERE imgname = ?")) {
    ps.setString(1, "myimage.gif");
    try (ResultSet rs = ps.executeQuery()) {
        while (rs.next()) {
            Blob blob = rs.getBlob(1);
            try (OutputStream os = blob.setBinaryStream(0)) {
                // Write data to the output stream. The output stream is buffered, so you don't need to
                // wrap it in a BufferedOutputStream
            } finally {
                blob.free();
            }
        }
    }
}

// Finally, commit the transaction.
conn.commit();
```

### Deleting a Large Object

Deleting a row from your table does **not** remove the underlying Large Object data. You must explicitly unlink it using `lo_unlink` before or after deleting the row:

```sql
SELECT lo_unlink(imgoid) FROM imageslo WHERE imgname = 'myimage.gif';
DELETE FROM imageslo WHERE imgname = 'myimage.gif';
```

<Accordion title="Using the LargeObjectManager API directly">

The driver also exposes a lower-level `org.postgresql.largeobject.LargeObjectManager` API (mirroring the libpq C interface) that provides random access to Large Object contents as if they were local files.

Access it via `PGConnection`:

```java
import org.postgresql.largeobject.LargeObject;
import org.postgresql.largeobject.LargeObjectManager;

conn.setAutoCommit(false);

LargeObjectManager lobj =
    conn.unwrap(org.postgresql.PGConnection.class).getLargeObjectAPI();

// Create a new large object
long oid = lobj.createLO(LargeObjectManager.READ | LargeObjectManager.WRITE);

// Open it for writing
LargeObject obj = lobj.open(oid, LargeObjectManager.WRITE);
byte[] buf = /* your data */;
obj.write(buf);
obj.close();

conn.commit();
```

Refer to the [Javadoc](http://javadoc.io/doc/org.postgresql/postgresql) for the full `LargeObjectManager` and `LargeObject` API.
</Accordion>
